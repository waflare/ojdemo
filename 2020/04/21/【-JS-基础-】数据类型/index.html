javascript 数据类型

  1. 六种原始数据类型

    • undefined

    • null

    • boolean

    • number

    • string

    • symbol（ES6 引入）

  2. 引用类型

    • Object 对象
      • 普通对象 Object
      • 数组对象 Array
      • 正则对象 RegExp
      • 日期对象 Date
      • 数学函数 Math
      • 函数对象 Function

二、原始数据类型

（1）undefined

未经初始化的变量默认值为 undefined，typeof 值为 undefined 的不一定是未经初始化，也可能是未声明。

var a;
alert(typeof a); // "undefined"
alert(typeof b); // 未声明，依旧是 "undefined"
alert(a); // "undefined"
alert(b); // error

（2）null

null 是对象吗？不是

var a = null;
alert(typeof a); // "object"

使用 typeof 检查 null 返回的值为 “object”，理所当然是一个 Object 对象，但是这只是 JS 存在的一个悠久 Bug。在 JS 的最初版本中使用的是 32 位系统，为了性能考虑使用低位存储变量的类型信息，000 开头代表是对象然而 null 表示为全零，所以将它错误的判断为 object 。

（3）boolean

数据类型 转换为true 转换为false
boolean true false
string 非空字符串 “”（空字符串）
number 非零数字值 0和NaN
object 任何对象 null
undefined N/A undefined

（4）number

  • 八进制

    以0开头，且有效的八进制数值

      var a = 070; // 有效八进制，解析为 56
  var b = 080; // 无效八进制，解析为 80

  • 十六进制

    以0x开头，且有效的十六进制数值

      var a = 0xA; // 有效十六进制，解析为10
  var b = 0xQ; // 无效十六进制，报 error

  • 浮点数值

    所谓浮点数就是该数值必须包含一个小数点，并且小数点后面必须至少有一位数字。浮点数值占用的空间是整型的两倍。

    • 科学记数法

        var a = 1.125e5; // 112500

    • 0.1 + 0.2 为什么不等于 0.3

      0.1和0.2在转换成二进制后会无限循环，由于标准位数的限制后面多余的位数会被截掉，此时就已经出现了精度的损失，相加后因浮点数小数位的限制而截断的二进制数字在转换为十进制就会变成0.30000000000000004。

  • 数值范围

    5e-324 < number < 1.7976931348623157e+308

    超过这个范围的数将自动转换为 Infinity 值，-Infinity（负无穷）或 +Infinity（正无穷），Infinity 值不参与运算。

  • NaN 非数值

    在很多语言里非数值参与数值运算会停止代码执行，但在 JS 里会返回 NaN 从而不影响后续代码运行。

      alert("adf"*2); // 结果为 NaN

    任何有 NaN 参与的数值运算结果都为 NaN。

    isNaN() 可以判断是否 “ 不是数值”

      alert(isNaN(NaN)); // true
  alert(isNaN(10)); // false 10是数值
  alert(isNaN("10")); // false 字符串“10”可以转换成数值
  alert(isNaN("blue")); // true 不能转换为数值
  alert(isNaN(true)); // false 可以转换为数值 1

  • 数值转换

    Number(), parseInt(), parseFloat()

（5）string

字符串创建后不可改变，字符串变量只能销毁后赋新值。

  • toString()

    可以带参数，表示以n进制形式输出

      var num = 10;
  alert(num.toString(2)); // "1010"

    如果是 null、undefined 不能使用该方法

  • String()

    • 有 toString() 方法的调用该方法
    • 如果是 null，返回 “null”
    • 如果是 undefined，返回 “undefined”

（6）symbol

它的功能类似于一种标识唯一性的ID。通常情况下，我们可以通过调用Symbol()函数来创建一个Symbol实例：

let s = Symbol();
typeof s; // "symbol"

上面代码中，变量s就是一个独一无二的值。

Symbol函数可以接受一个字符串作为参数，表示对 Symbol 实例的描述，主要是为了在控制台显示，或者转为字符串时，比较容易区分。

let s1 = Symbol('foo');
let s2 = Symbol('bar');

s1 // Symbol(foo)
s2 // Symbol(bar)

s1.toString() // "Symbol(foo)"
s2.toString() // "Symbol(bar)"

Symbol 的参数也可以是一个对象，就会调用该对象的toString方法，将其转为字符串，然后才生成一个 Symbol 值。

const obj = {
  toString() {
    return 'abc';
  }
};
const sym = Symbol(obj);
sym // Symbol(abc)

有人会想 Symbol 的参数相同的是不是返回一样的值

// 没有参数的情况
let s1 = Symbol();
let s2 = Symbol();

s1 === s2 // false

// 有参数的情况
let s1 = Symbol('foo');
let s2 = Symbol('foo');

s1 === s2 // false

很明显，结果是没有参数或者参数相同返回值都是不一样的。

Symbol 不能参与运算，可以显式转为字符串，也可以转为布尔值，但是不能转为数值。

Symbol 还有 description 属性显示 Symbol 描述

const sym = Symbol('foo');
sym.description // "foo"

有时，我们希望重新使用同一个 Symbol 值

  • Symbol.for()

      let s1 = Symbol.for('foo');
  let s2 = Symbol.for('foo');

  s1 === s2 // true

    Symbol.for()与Symbol()这两种写法，都会生成新的 Symbol。它们的区别是，前者会被登记在全局环境中供搜索，后者不会。Symbol.for()不会每次调用就返回一个新的 Symbol 类型的值，而是会先检查给定的key是否已经存在，如果不存在才会新建一个值。比如，如果你调用Symbol.for("cat")30 次，每次都会返回同一个 Symbol 值，但是调用Symbol("cat")30 次，会返回 30 个不同的 Symbol 值。

返回一个已登记的 Symbol 类型值的key。

  • Symbol.keyFor()

      let s1 = Symbol.for("foo");
  Symbol.keyFor(s1) // "foo"

  let s2 = Symbol("foo");
  Symbol.keyFor(s2) // undefined 未登记在全局环境中

注意，Symbol.for()为 Symbol 值登记的名字，是全局环境的，不管有没有在全局环境运行。
